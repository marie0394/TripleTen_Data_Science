SEED = 33333
def upsample(data, target, repeat):
    data_zeros = data[data[target] == 0]
    data_ones = data[data[target] == 1]
    
    repeat_int = int(repeat)
    repeat_dec = repeat - repeat_int
    
 

    data_upsampled = pd.concat([data_zeros] +
                               [data_ones] * repeat_int +
                               [data_ones[0:int(repeat_dec*len(data_ones))]])


    data_upsampled = shuffle(data_upsampled, random_state=SEED)

    return data_upsampled

def find_metrics(y_true, y_pred, title = 'Mtricas'):
    accuracy = accuracy_score(y_true, y_pred)
    roc_auc = roc_auc_score(y_true, y_pred)
    df = pd.DataFrame(data={'Precision': [accuracy], 
                           'ROC-AUC': [roc_auc]})
    df.columns = pd.MultiIndex.from_product([[title], df.columns])
    # df.style.set_caption(title)
    display(df)
    return [accuracy, roc_auc]

Nos interesa el valor de la mtrica, pero tambin nos interesa que el modelo sea confiable para predecir. Podramos decir que el modelo 4, CatBoost, sera el elegido, ya que fue el que result con el mejor ROC-AUC. Sin embargo, tambin tiene la mayor diferencia entre los resultados del dataset de prueba vs el entrenamiento. Indicara que es el modelo con mayor problema de sobreajuste de los probados.

Dado esto, la eleccin final sera con el modelo 2, que us bosque aleatorio, ya que di un ROC-AUC de 0.896, el segundo ms alto, teniendo tambin de los menores sobreajustes, lo que nos da ms confianza de que el modelo sirva para predecir datos nuevos.

Conclusiones¶

Se comenz con 4 archivos con informacin de los contratos y los clientes, los cuales fueron unidos a travs del CustomerID. Se realizaron cambios a los tipos de variables de un par de columnas antes de continuar con la unin. Se cre la variables ContractEnded la cual sera nuestra variablle objetivo.

Luego se realiz el preprocesamiento, primero se rellenaron los datos ausentes. Como estos valores ausentes provienieron de las tablas de los planes de internet y telfono, se asumi que esto implicaba que no estba usando dicho servicio un usuario en especfico. Por tanto, se rellen con 'No' estas variables para indicar que el usuario no tiene esos servicios activos. Luego se calcul una variable extra que contendra el nmero de meses que ha tenido un usuario el contrato.

La mayora de datos eran categricos binarios, por lo que fueron reemplazados por valores de 1/0. Luego se quitaron las columnas que no seran necesarias para el entrenamiento. Dado que la relacin entre la clase positiva y negativa de la funcin objetivo estaba desbalanceada, se realiz sobremuestreo, para equipar mejor la proporcin de ambos.

Varios modelos requieren solo usar datos numricos por lo que se aplic One-Hot Encoding para tratar los valores categricos restantes.

Adems de la prueba de cordura, se crearon 4 modelos, uno que aplic regresin logstica, otro de bosque aleatorio, uno con LightGBM y otro de CatBoost. Los 4 superaron la prueba de cordura y los mejores valors de AUC-ROC se obtuvieron del modelo 2 y 4. Aunque el modelo 4 tena la mejor mtrica, tambin presenta el mayor sobreajuste. Por tanto, se eligi el modelo 2, que tiene aun as el segundo mejor AUC-ROC pero tiene menor sobreajuste, dando ms confianza que este modelo pueda predecir mejor en casos de datos nuevos.
